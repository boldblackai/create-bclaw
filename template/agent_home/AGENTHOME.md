# agent_home/

This directory is the **source of truth for curated claw state**. The
`manage-bclaw` skill overlays its contents 1:1 onto the running
claw's `/home/harness/.hermes/` (EFS-backed).

## What goes here

Mirror the structure you want on the claw. Only include files you intend to
control — everything else on the claw is left untouched (overlay, not replace):

```
agent_home/
  config.yaml          # non-env-driven config (skills, toolsets, MCP, model)
  system-prompt.md     # custom system prompt / persona
  skills/              # curated skills (SKILL.md + references/)
  memories/            # curated memories
  personas/            # persona definitions
```

## What NOT to put here

Runtime-generated or large data — it gets tarred and shipped over ECS Exec on
every update:

- `sessions/` — live conversation state (generated by the claw)
- `.cache/` — model caches, HF hub downloads (large, regenerated automatically)
- `~/.config/gh` — GitHub credentials (managed by on-boot `gh auth login`, when `EnableGitHubKey=true`)
- Model weights, binaries, or anything measured in MB+

## How it's applied

See `.agents/skills/manage-bclaw/SKILL.md`. The overlay is a merge
with overwrite: new files are added, changed files are overwritten, and files
absent from here are preserved on the claw.
